### Objective
Observed application architecture and communication flow using Burp Suite and Browser DevTools.

### Findings
- Application behaves like a Single Page Application (SPA) because the URL changes without full page reload and content updates dynamically
- Multiple JavaScript chunk files observed which indicates bundled frontend modules and modern JavaScript framework usage
- REST API endpoints identified, indicating frontend-backend API communication
- JSON responses observed, showing structured data exchange between client and server
- `socket.io` references identified, indicating possible real-time communication support
- Wappalyzer identified Angular framework usage (`Angular 21.2.12`)

### Security-Relevant Observations
- API-driven architecture may increase importance of API authorization testing
- SPA behavior suggests client-side routing and dynamic rendering
- JavaScript bundles may contain hidden routes, API references, or frontend logic
- `socket.io` may introduce websocket-related attack surfaces
- Framework and version information may help guide future framework-specific security research

### Learning Outcomes
Improved understanding of:
- Frontend framework fingerprinting
- REST API communication
- JSON-based client-server interaction

### Notes
This phase focused only on reconnaissance and architecture observation.
No vulnerability testing or exploitation was performed during this phase.
