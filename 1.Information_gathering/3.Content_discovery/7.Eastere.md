## eastere.gg

### Observation
A hidden file was discovered through content discovery.

### Access Method
- Direct access to the file was restricted by the application's extension whitelist.
- The file was retrieved by appending a double-encoded null byte followed by an allowed extension:
- <img width="700" alt="image" src="https://github.com/user-attachments/assets/88445d01-3b0a-460b-97c0-a0472e35e34f" />
```text
eastere.gg%2500.md
```
- <img width="1918" height="613" alt="image" src="https://github.com/user-attachments/assets/03128227-db43-49e2-aa22-f4bf2f888a48" />
This satisfied the extension validation logic and allowed the original file contents to be returned.

### Analysis
The file contained a Base64-encoded string.
- <img width="600" alt="image" src="https://github.com/user-attachments/assets/b442e126-10f6-4611-8546-647a0f9ec027" />
#### Step 1 - Base64 Decode
- <img width="600" alt="image" src="https://github.com/user-attachments/assets/01d31143-50f2-4964-ae06-994e43f66ed1" />

```text
/gur/qrif/ner/fb/shaal/gurl/uvq/na/rnfgre/rtt/jvguva/gur/rnfgre/rtt
```

#### Step 2 - ROT13 Decode
```text
/the/devs/are/so/funny/they/hid/an/easter/egg/within/the/easter/egg
```

### Findings
The decoded message states:
```text
The devs are so funny they hid an easter egg within the easter egg.
```

### Why Interesting?
The file demonstrates that:
- Hidden content may exist within publicly accessible directories.
- Multiple layers of encoding may be used to conceal information.

### Recon Takeaway
Content discovery may reveal hidden files containing encoded clues. Multiple layers of encoding (Base64 + ROT13) were used to conceal the message,
demonstrating the importance of analyzing unusual file contents rather than relying solely on filenames or extensions.
