## coupons_2013.md.bak

### Initial Observation
A backup file (`.bak`) was identified within the `/ftp/` directory.

### Recon Process
#### Step 1: Direct Access Attempt
Request:
```text
http://localhost:3010/ftp/coupons_2013.md.bak
```
Response:
```text
403 Forbidden
```
- <img width="600" alt="image" src="https://github.com/user-attachments/assets/f0c9785c-49a4-4ecf-9473-2f34df6e4d85" />
The application enforced an extension whitelist and denied access.

#### Step 2: Extension Whitelist Observation
- Previous testing of the file server showed that only `.md` and `.pdf` files were permitted.
- The backup file did not match the allowed extensions.

#### Step 3: Double-Encoded Null Byte Bypass
Request:
```text
http://localhost:3010/ftp/coupons_2013.md.bak%2500.md
```
- The payload satisfied the extension validation while allowing access to the original filename.

#### Step 4: File Disclosure
The request returned file contents successfully.

### Content Analysis
Observations:
- Content is not human-readable.
- Multiple entries share a repeating suffix
- Data may be encrypted, encoded, or obfuscated.

### Why Interesting?
Backup files frequently contain:
- Historical application data
- Previous versions of files
- Business information
- Legacy coupon data
- Credentials or configuration artifacts
Even when the contents are not immediately readable, backup files can provide useful intelligence about application functionality.

### Recon Takeaway
A backup file was accessible through a file extension validation bypass.
Although the contents were not directly readable, the file demonstrates that historical application artifacts can remain exposed on 
web servers and may contain information requiring additional analysis.
