# package-lock.json Information Disclosure

### Observation
- A backup copy of `package-lock.json` was accessible through the `/ftp/` directory.
- The file disclosed detailed dependency information used by the application.

### Access Method
- Direct access to the backup file returned a `403` response, revealing that only `.md` and `.pdf` files were permitted by the application's
  extension whitelist.
- <img width="600" alt="image" src="https://github.com/user-attachments/assets/c48aab9a-0a3e-443d-b142-3a72c894d81f" />
- Appending an allowed extension (`package-lock.json.bak.md`) resulted in a `404` response, indicating that the extension validation was bypassed,
  but the file did not exist under that modified filename.
- A null byte payload was used to bypass the extension validation, but the application returned a `404 Not Found` response and the file contents were
  not disclosed.
- A double-encoded null byte payload was then appended to the original filename:
```text
package-lock.json.bak%2500.md
```
- The payload satisfied the extension whitelist while allowing the original file contents to be returned.

### Findings
- <img width="600" alt="image" src="https://github.com/user-attachments/assets/7263ba0f-a4c6-4e4f-9b00-1aab56241f3d" />
- <img width="600" alt="image" src="https://github.com/user-attachments/assets/af03d1e2-fe76-4d3b-bab8-7725b44c4a33" />
- **Application Information**: Identifies the application name, version, and package manager metadata.
- **Direct Dependencies**: Reveals packages directly used by the application.
- **Transitive Dependencies**: Reveals additional packages installed automatically by other dependencies.
- **Exact Version Information**: Reveals the exact installed version of each package instead of a version range.
- <img width="600" alt="image" src="https://github.com/user-attachments/assets/d51b1960-7813-47eb-9a73-dccebd6587f3" />
- **Integrity Information**: Contains hashes used to verify package integrity and ensure packages were not modified during installation.

### Recon Perspective
```text
package.json
- Shows what the developers intended to use.
- Tech stack
- Direct dependencies
- Runtime requirements

package-lock.json
- Exact versions
- Transitive dependencies
- Dependency tree
- Integrity information
```

### Analysis
Exposed package-lock files may reveal:
- Exact dependency versions
- Hidden transitive dependencies
- Package integrity information
- Potentially vulnerable libraries
- Additional attack surface not visible in package.json

### Recon Takeaway
An exposed `package-lock.json` is often more valuable than `package.json` because it reveals the exact packages installed by the application. 
This allows more accurate version enumeration, dependency analysis, and vulnerability research against both direct and transitive dependencies.
